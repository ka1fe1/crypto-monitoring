## Findings

### P0 - Critical

无

### P1 - High

1. **BP 获取失败使用 fmt.Printf 而非 logger**

- 影响/场景：错误输出未使用统一的日志框架，无法被日志收集系统捕获
- 定位：[`btc_dashboard_service.go:L180`](../../internal/service/btc_dashboard_service.go#L180)
- 建议：使用项目统一的 logger：
```go
logger.Warn("failed to fetch balanced price: %v", err)
```

### P2 - Medium

1. **BP 阈值常量命名与实际语义可能不符**

- 影响/场景：`bpRatioThresholdNormal = 2.0` 被定义为"正常区间"上限，但报告中显示 > 2.0 为"过高区间"
- 定位：[`btc_dashboard_service.go:L26`](../../internal/service/btc_dashboard_service.go#L26)
- 建议：考虑将常量重命名为更具描述性的名称，如 `bpRatioThresholdOvervalued`

2. **bgeometrics client 缺少超时配置灵活性**

- 影响/场景：HTTP 客户端超时硬编码为 10 秒，无法根据网络环境调整
- 定位：[`bgeometrics/bgeometrics.go:L22-L24`](../../pkg/utils/bgeometrics/bgeometrics.go#L22-L24)
- 建议：考虑从 config 读取超时配置，或提供 WithTimeout 选项模式

3. **API Key 为空时的行为**

- 影响/场景：当 API Key 为空时，请求仍会发送（只是不带 key 参数），可能导致 401 错误
- 定位：[`bgeometrics/bgeometrics.go:L36-L40`](../../pkg/utils/bgeometrics/bgeometrics.go#L36-L40)
- 建议：考虑在 NewClient 时检查 API Key 是否为空，或在 GetBalancedPrice 中提前返回错误

### P3 - Low

1. **测试跳过条件过于严格**

- 影响/场景：bgeometrics 测试需要真实 API Key，无法在无 key 环境下验证 client 逻辑
- 定位：[`bgeometrics/bgeometrics_test.go:L42-L44`](../../pkg/utils/bgeometrics/bgeometrics_test.go#L42-L44)
- 建议：可以添加一个 mock server 测试，验证解析逻辑而不依赖真实 API

2. **响应解析双重 fallback 逻辑复杂**

- 影响/场景：代码尝试两种格式解析，增加了维护成本
- 定位：[`bgeometrics/bgeometrics.go:L54-L64`](../../pkg/utils/bgeometrics/bgeometrics.go#L54-L64)
- 建议：如果 API 格式稳定，建议只保留一种解析方式；或添加注释说明为何需要双重解析
