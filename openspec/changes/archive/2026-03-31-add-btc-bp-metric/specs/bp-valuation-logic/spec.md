## ADDED Requirements

### Requirement: 对 Balanced Price 的比例进行区间划分与估值推送

在已知 BP 的情况下，提供逻辑：当前价格相比均衡价格有怎样的溢价或折价倍数，并在输出消息上对其落入何种象限进行判定。

#### Scenario: 当前价格除以 BP 获取倍率
- **WHEN** 从数据源取到了比特币的当前市场价格和 Balanced Price。
- **THEN** 需要除以 BP 得到当前价 / 均衡价的比率，四舍五入保留至少两位小数（例如 1.67x）。

#### Scenario: 判断当前估值空间
- **WHEN** 计算出了倍率后（如 screenshot 中的 1.67x）。
- **THEN** 应用以下经验性阈值判定其状态名称及图表前缀，展示格式上应当符合“当前价格 / BP = xxx”，并在下一行标注如 `📈 高于 BP — 正常估值区间` 或者 `📉 低于 BP — 历史大底（被极度低估）` 等可视化文案。
